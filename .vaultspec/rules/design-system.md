# Design system: Figma-binding, centralized, tokenized

- **Figma is the binding source of truth.** File `SlhonORmySdoSMTQgDWw3w` binds the design — foundation (color, type, spacing, radius, elevation), the Kit, surface frames, the node canvas. Code mirrors Figma; on disagreement Figma wins and code is corrected. Any deliberate divergence (a font substitution, a platform constraint) requires an ADR recording it.
- **The design system is centralized.** Every UI element derives from the one system: a base component instance, a bound color variable, a shared text style, or a shared elevation style. Never redefine a primitive ad-hoc (hand-built button/chip/tab/switch/field, raw hex, loose `fontSize`/weight, bespoke shadow). Bespoke composites (graph, timeline, diff, menu rows) compose the standardized atoms. A missing primitive is a gap to close in the library, not a license to inline.
- **Themes are OKLCH ramps through one semantic tier.** Colors derive from primitive OKLCH ramps aliased by one semantic token tier, emitted on `:root` for DOM + scene. A theme is a `[data-theme]` remap of that tier; high-contrast is another remap — no component knows the active theme. Scene-read tokens (consumed via `getComputedStyle`) MUST be emitted as literal hex per theme, never `var()` aliases (the reader won't flatten a var chain).
- **Warmth lives in tokens, not decoration:** warm low-chroma neutrals, one muted accent, soft depth/radius, purposeful micro-interactions, copy tone. Never textures, skeuomorphism, gradients, multiple accents, or reduced contrast. Contrast, diff legibility, density, and reactivity override warmth on any conflict.
- **Icons from two families only.** Structural chrome from Lucide; expressive/domain marks from Phosphor (or authored in-family on Phosphor's grid). No third set. Every domain mark passes the 14px ink-coverage grayscale gate (`svgRaster.ts` + `markGate.ts`) — a winding-rule/stroke-band raster Hamming comparison, never a geometric containment test.
- **No hardcoded px.** Every size/spacing/padding/margin/radius/border/shadow/type value is `rem` at the 16px basis (or `em`), routed through the DTCG `--*-fg-*` token scale where one fits. Enforced by `lint:px` (`frontend/scripts/scan-px.mjs`); allowlist stays empty. Convert off-scale values value-preserving (rem = px÷16), never snapped to a nearer token. Exceptions: the WebGL scene (bridges via `uiScale()` = rootFontPx/16), test fixtures, `prototype`, CLI-managed token regions.
- **Labels are user-facing.** Every rendered string is plain language. Internal vocabulary (seam/param names, physics/graph-theory terms, backend/tier ids, wire field names) may live in source but never on screen. A control binding an internal param renders a reworded label; the param key keeps its internal name.
- **The Figma↔code join is name-as-contract:** a component's Figma node name EQUALS its React export (bare PascalCase). The only fallback is an opt-in `// @figma <Name> · <fileKey> · <nodeId>[ · alias-of <Export>]` header, validated by `figma:names` (in `just dev lint frontend`). No Code Connect, no `component-map.json`, no `.figma.tsx`.
- **Settings are schema-driven from one registry.** Every setting is declared once in the engine registry (`engine/crates/vaultspec-session/src/settings_schema.rs`); validation, the served `/settings/schema`, effective-value resolution, and the rendered control all derive from it. Never hand-wire a setting outside the registry, and never ship a control persisting a value nothing reads.
