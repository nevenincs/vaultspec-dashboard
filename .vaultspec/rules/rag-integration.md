# rag integration: machine singleton, codified contract

- **rag is one resident service per machine; the dashboard attaches, never owns.** Authority is the OS machine lock. Determine "running" from machine-global discovery (`~/.vaultspec-rag/service.json`) + heartbeat freshness + an ungated `GET /health` `status=="ready"` with a live pid. Manage whatever service is running regardless of who started it; start your own only when one is genuinely absent — never speculatively, never per-workspace. Gate `server start` on that predicate: treat exit 0 as started and a non-zero already-running result as the signal to re-discover and ATTACH (never 502). Pass no `--json` to `server start`/`server stop` (rag 0.2.25 rejects it).
- **Never override rag's `STATUS_DIR`.** The dashboard never sets `VAULTSPEC_RAG_STATUS_DIR`; discovery stays machine-global with the home path (`~/.vaultspec-rag/service.json`) winning over any per-scope file, and the lifecycle spawn carries no STATUS_DIR. A per-scope override fragments discovery while the machine lock still allows one service.
- **Read rag through its codified contract, never its Qdrant shape.** Read size/state/data/diagnostics from rag's HTTP control plane (Tier 1: `/service-state`, `/storage/survey`, `/jobs`, `/projects`, `/metrics`, `/health`, `/readiness`). For only what rag doesn't expose (optimizer/segment/indexed-vs-total health), read Qdrant's own documented REST API (Tier 2), capability-gated on the Qdrant version, with collection names from `/storage/survey`. Never derive a value from rag's internal collection naming (the blake2b prefix) or payload shape. Degrade the affected panel fail-closed on a capability/version mismatch.
  - SANCTIONED EXCEPTION (2026-07-02, RCR-003 in `2026-07-02-rag-console-review-audit`; decided by the `graph-semantic-embeddings` ADR D1, which PREDATES the survey route): the embedding-scroll path (`rag-client/src/vectors.rs` `vault_collection_name`, consumed by the `/graph/embeddings` route) recomputes the blake2b-6 vault-collection prefix. It is tolerable ONLY under its existing conditions — the rag-schema-gate version-gates the dense vector name/dim (though NOT the naming scheme itself), and a naming-scheme change degrades HONESTLY (404 → no vectors, never wrong data). No new recompute site may cite this exception. SUNSET: the exception retires the moment a codified per-root collection-name source exists (the survey exposing the scroll-time vault collection name, or a rag control-plane route) — that gap is a Tier-3 coordination ask to FILE toward rag, not to patch around; when it lands, re-source the scroll and delete this clause.
