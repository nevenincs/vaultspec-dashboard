# Actions, keymap, palette: one descriptor across eligible planes

- **One `ActionDescriptor` per action** (`frontend/src/platform/actions/action.ts`), enrolled unchanged across the planes it is ELIGIBLE for under ONE shared action `id`: the per-kind context-menu resolver registry (`platform/actions/registry.ts` + `app/menus/registerAll.ts`), the keymap registry + dispatcher (`platform/keymap/registry.ts` + `stores/view/keymapDispatcher.ts`), the command-palette providers, and the dispatch seam (`appDispatcher` → `opsActions`/`sessionActions`/store mutators). A verb recurring across surfaces is authored once in a shared builder (`app/menus/sharedActions.ts`, `stores/view/graphCommands.ts`, `stores/view/chromeActions.ts`) and composed. Never a bespoke per-surface handler; never re-author an action another surface defines. Enroll a verb only where eligible; remove a non-capability rather than ship a permanently-disabled lie.
- **Context-menu actions live on exactly one of three layers:** the bespoke per-kind resolver (payload depends on the clicked entity), the one global tail (`registerGlobalTailActions`, identical for every target, capped at Refresh, terminal `global` section), or the background empty-space menu (`app/menus/backgroundMenu.ts` — chrome escape hatches + Refresh). Never duplicate a verb across layers.
- **Command shortcuts bind through the one keymap registry.** Every Class-A command shortcut is one `KeybindingDef` fired by the single global dispatcher. No `app/`/`scene/` surface grows its own global `keydown` for a command. Customization is only the engine `keybindings` setting (bounded sparse `action_id→chord` map, `MAX_KEYBINDING_OVERRIDES`, validated). Class-B widget-intrinsic ARIA keys (roving tabindex, focus trap, dismiss-on-escape, menu/listbox cursoring) stay in the component; never route through the registry.
- **Composites navigate through the one `FocusZone`** (`app/chrome/useFocusZone.ts`): every multi-child interactive composite contributes exactly one tab stop (active child `tabIndex=0`, siblings `-1`; arrows/Home/End rove) and `stopPropagation`s consumed keys so they never reach the global dispatcher. Inter-region traversal (F6) is a keymap command, never a private global listener. Never hand-roll a roving-tabindex loop.
- **Cmd+K commands come from the one provider registry.** Every command is a pure `CommandProvider` registered in `stores/view/commandRegistry.ts`, consumed via `resolveCommands`. No hand-assembled command list. Never enroll transient corpus data (per-document/feature/lens) as a standing command — corpus navigation is the document-search plane only.
- **Palette accelerators derive from the keymap registry** by shared `id` (`deriveCommandAccelerators`). Never hand-type an accelerator or maintain a second legend list; the `?` legend derives from the registry.
- **One open verb for every result entity.** Opening a search hit, graph node, or menu target uses the shared `openEntityAction` over the `openNodeIsland` seam. No surface re-implements open/go-to.
- Guards: `actionCoverage.guard.test.ts`, `commandPalette.guard.test.ts`, `chromeActions.test.ts`, `backgroundContextMenu.test.ts`.
