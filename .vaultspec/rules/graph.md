# Graph: bounded reads, CPU compute, pinned canvas, one snapshot

- **Every graph read is bounded.** Constellation (feature) LOD is the unbounded-safe default. Document granularity is always served under the `MAX_DOCUMENT_NODES` ceiling, kept self-consistent (only edges among kept nodes), with any truncation stated in a `truncated` block. Never serialize an unbounded full-document slice. Descend into detail only scoped — by feature filter or ego/neighbor query.
- **Compute is CPU; GPU is render + search.** Graph construction, diff, projection, and query in `engine/` stay CPU-bound. GPU serves only rendering (`frontend/src/scene/`) and semantic search (rag over HTTP). Scale via linear ingest, bounded payloads, LOD, and memoized projections — never by moving the engine onto a GPU. Engine crates carry no CUDA/torch/wgpu dependency.
- **Filter facets apply on the engine.** Every reducing facet (`doc_types`, `statuses`, `plan_tiers`, `text`, `feature_tags`, `feature_query`, `tiers`, `relations`, `structural_state`, `min_confidence`, `health`, `date_range`, `kinds`) is applied in `/graph/query`, carried by `dashboardGraphFilter` and the query key. Never re-derive a facet as a client narrow of the served slice (feature-aggregation and the node ceiling both make it silently wrong). `computeVisibility` narrows only client-held data: working-set ego expansions, pinned discoveries, the legend category mask. Smoothness comes from `placeholderData: keepPreviousData` + the bounded cache.
- **Derived projections memoize on `generation`.** Every per-request projection over `LinkageGraph` (enriched node/edge views, constellation `meta_edges`, salience basis) is cached keyed on the immutable graph `generation` and invalidated on commit. The graph swap happens-before the generation bump (both `SeqCst`), so a reader on `G+1` sees the new graph.
- **The canvas is portal-pinned, never re-parented.** The Pixi/three `<canvas>` mounts once for the app lifetime in a host that is a SIBLING of any dock/layout container, never inside it. A layout library owns only a placeholder tracked by rect (`canvasPin`). Never re-parent or remount the canvas node — it destroys the WebGL context and the live `SceneController`.
- **Settled layout is pin-authoritative, not force-converged.** The rest state freezes at `alphaMin` and pins every node. Inject energy only through named gentle entry points: `set-data` prewarm, change-proportional `reheatGentle`, explicit `reheatNow()`/cold restart. Never a bare global unpin (`wakeAllFree`) + WARM reheat. `resume`/`pause` are energy-neutral. Every ambient warm data path (ego expansion, live delta splice, same-scope re-fetch) pins carried survivors before relaxing (`prewarmReflow`); a same-id-set update ticks zero.
- **Present-view reads one corpus snapshot.** Live graph sources NODES and declared `related:` EDGES from the SAME snapshot — the working tree (`vault graph`, no `--ref`), via a document-read-only subprocess. Only historical/as-of views read a committed `--ref <sha>`. The present-view declared-fold cache keys on `worktree_corpus_fingerprint` (content of in-scope docs), never the HEAD sha. Keep present-view (fingerprint) and as-of (sha) cache key spaces distinct.
